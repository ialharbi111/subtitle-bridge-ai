# إضافة Stremio: ترجمة عربية تلقائية بالذكاء الاصطناعي (Gemini)

إضافة تجلب ملف الترجمة الإنجليزية المتوافق تماماً مع نسخة الفيديو المشغَّلة (عبر مطابقة الـ hash)، وتترجمه فوراً إلى العربية الفصحى باستخدام Gemini، مع الحفاظ الكامل على التوقيت، وذاكرة مصطلحات تحفظ أسماء الشخصيات وأسلوب الترجمة عبر حلقات نفس المسلسل.

---

## 1. آلية العمل باختصار

1. **المطابقة**: عند اختيار المستخدم لحلقة/فيلم، تجلب الإضافة الترجمة الإنجليزية من OpenSubtitles، وتحاول أولاً مطابقتها عبر `moviehash` (يرسله Stremio تلقائياً) لضمان تطابق التوقيت مع النسخة المشغَّلة بالضبط. إن لم تجد تطابق hash، تلجأ لأفضل نتيجة متاحة حسب IMDB ID.
2. **الحماية من كسر التوقيت**: الكود **لا يرسل التوقيت لـ Gemini إطلاقاً** — يتم تفكيك ملف الـ SRT إلى نصوص فقط، ثم إعادة تركيبها بنفس التوقيت الأصلي 100%. هذا أضمن بكثير من مجرد "الطلب" من الذكاء الاصطناعي عدم لمس التوقيت.
3. **الترجمة**: يُرسل محتوى النصوص إلى `gemini-1.5-flash` بصيغة JSON، على دفعات (Batches) تُعالَج بالتوازي لتحقيق سرعة عالية.
4. **الكاش**: أي حلقة/فيلم تُترجم مرة واحدة يُحفظ ملفها كـ `.srt` على القرص، وتُخزَّن بيانات الوصول إليه في SQLite. الطلبات اللاحقة لنفس الحلقة تُقرأ فوراً دون أي استدعاء لـ Gemini.
5. **ذاكرة الاتساق (Glossary)**: بعد كل ترجمة، تقوم الإضافة في الخلفية (دون تأخير الاستجابة) باستخراج أسماء الشخصيات والمصطلحات المتكررة وحفظها في SQLite مرتبطة بـ IMDB ID الخاص بالمسلسل. عند ترجمة الحلقة التالية، تُرسل هذه القائمة إلى Gemini كجزء من التعليمات ليلتزم بنفس الترجمات.

## 2. هيكل الملفات

```
stremio-ai-subtitles/
├── index.js                # نقطة الدخول: manifest + معالج subtitles + سيرفر Express
├── package.json
├── .env.example             # انسخه إلى .env وضع مفاتيحك هنا
├── src/
│   ├── opensubtitles.js     # جلب الترجمة الإنجليزية (بحث + تحميل + مطابقة hash)
│   ├── translator.js        # الاتصال بـ Gemini + التجزئة + ذاكرة المصطلحات
│   ├── srtParser.js         # تفكيك/إعادة بناء SRT بأمان (حماية التوقيت)
│   └── db.js                 # SQLite: الكاش + ذاكرة المصطلحات
├── data/                     # (يُنشأ تلقائياً) قاعدة بيانات SQLite
└── cache_files/               # (يُنشأ تلقائياً) ملفات الـ SRT العربية الجاهزة
```

## 3. أين تضع مفاتيح الـ API

1. انسخ `.env.example` إلى ملف جديد اسمه `.env`:
   ```bash
   cp .env.example .env
   ```
2. افتح `.env` وضع:
   - **`GEMINI_API_KEY`**: احصل عليه مجاناً من [Google AI Studio](https://aistudio.google.com/apikey).
   - **`OPENSUBTITLES_API_KEY`**: سجّل حساباً مجانياً على [opensubtitles.com](https://www.opensubtitles.com/en/consumers) ثم أنشئ API Key من لوحة تحكم المطورين.
   - (اختياري) `OPENSUBTITLES_USERNAME` و `OPENSUBTITLES_PASSWORD`: تسجيل الدخول يرفع الحد اليومي للتحميل من OpenSubtitles (الحساب المجهول محدود جداً، ~5 تحميلات/يوم فقط).

⚠️ **لا ترفع ملف `.env` إلى GitHub أبداً** — هو مُستبعد تلقائياً عبر `.gitignore`.

## 4. التشغيل المحلي

```bash
cd stremio-ai-subtitles
npm install
npm start
```

سترى في الطرفية:
```
🚀 الإضافة تعمل على المنفذ 7000
📋 رابط الـ manifest: http://127.0.0.1:7000/manifest.json
```

### إضافتها في Stremio (اختبار محلي):
1. افتح تطبيق Stremio.
2. اذهب إلى **الإضافات (Addons)** > **إضافة أخرى عبر الرابط**.
3. الصق: `http://127.0.0.1:7000/manifest.json`
4. شغّل أي فيلم/حلقة له IMDB ID، وستجد خيار الترجمة العربية باسم "ترجمة عربية بالذكاء الاصطناعي" ضمن قائمة الترجمات.

> ملاحظة: التشغيل المحلي يعمل فقط على نفس الجهاز أو الشبكة المحلية. لاستخدامها من أي مكان (خصوصاً على الجوال أو Stremio على جهاز آخر)، يجب نشرها كما في القسم التالي.

## 5. النشر (Deployment) لجعلها متاحة دائماً

أسهل خيار مجاني هو **Render.com** أو **Railway.app**. الخطوات متشابهة:

### عبر Render:
1. ارفع المجلد إلى مستودع GitHub جديد (تأكد أن `.env` غير مرفوع).
2. من لوحة Render: **New > Web Service** واربطه بالمستودع.
3. Build Command: `npm install`
4. Start Command: `npm start`
5. أضف متغيرات البيئة (Environment Variables) من إعدادات الخدمة: `GEMINI_API_KEY`, `OPENSUBTITLES_API_KEY`, وبعد أول نشر انسخ الرابط الذي يعطيك إياه Render (مثل `https://your-app.onrender.com`) وضعه في متغير `BASE_URL`.
6. **مهم**: خطط Render المجانية تملك تخزيناً مؤقتاً (ephemeral) يُمسح عند إعادة التشغيل، فقاعدة SQLite وملفات الكاش قد تُفقد. لحل هذا أضف "Persistent Disk" (متاح في الخطط المدفوعة) وأشّر مساره إلى مجلد المشروع، أو استخدم Railway الذي يوفر تخزيناً مستمراً بسهولة أكبر.

### عبر VPS خاص (الأكثر استقراراً):
```bash
git clone <رابط مستودعك>
cd stremio-ai-subtitles
npm install
cp .env.example .env   # ثم عدّل القيم
npm install -g pm2
pm2 start index.js --name stremio-ar-subs
pm2 save
```
ثم استخدم Nginx كـ reverse proxy مع شهادة SSL (عبر Certbot) حتى يكون الرابط `https://your-domain.com/manifest.json` — Stremio يفضّل الروابط الآمنة (HTTPS) للإضافات المنشورة.

بعد النشر، ضع رابط الـ manifest النهائي (`https://.../manifest.json`) في Stremio بنفس طريقة الإضافة المحلية.

## 6. نقاط مهمة ومحاذير

- **الزمن المستهدف (<40 ثانية)**: يعتمد على طول الحلقة. الحلقات العادية (~800-1200 سطر حوار) تُترجم عادة خلال 15-30 ثانية بفضل التوازي بين الدفعات (Batches). الأفلام الطويلة جداً قد تحتاج وقتاً أطول قليلاً؛ يمكنك تقليل `BATCH_SIZE` في `src/translator.js` لزيادة عدد الدفعات المتوازية إن رغبت بسرعة أعلى (على حساب استهلاك أكبر لحصة الـ API).
- **حدود Gemini المجانية**: لخطة `gemini-1.5-flash` المجانية حدود لعدد الطلبات بالدقيقة/اليوم. إن كنت تستخدمها بكثافة، فعّل الفوترة على مشروع Google AI Studio لتفادي رسائل rate-limit.
- **حدود OpenSubtitles**: الحساب المجهول محدود جداً بالتحميلات اليومية؛ استخدام حساب مسجّل (username/password في `.env`) يرفع الحد بشكل ملحوظ.
- **حقوق الترجمات**: ملفات الترجمة الإنجليزية الأصلية مملوكة لرافعيها على OpenSubtitles؛ هذه الإضافة تُنشئ نسخة مترجمة للاستخدام الشخصي، فيرجى الالتزام بشروط استخدام OpenSubtitles عند الاستخدام أو النشر العام.
- **تحسين لاحق ممكن**: إضافة تنظيف دوري (cron) لحذف ملفات `cache_files` القديمة جداً إن أردت التحكم بحجم التخزين على المدى الطويل.
